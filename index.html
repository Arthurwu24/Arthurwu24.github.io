聊聊直播和短视频(1)

发表于 2017-12-23 | 分类于 产品分析 |
字数统计: 2,202 字 | 阅读时长 ≈ 7 分钟

这两年来，中国的风口是从直播吹到了短视频。

下面就说说我个人这几年来对直播和短视频的理解和看法。

直播

直播是在15年开始兴起的，但在16年迎来了“网络直播元年”，彻底引爆了市场。

  • 映客2年估值70亿

  • 陌陌在切入直播领域后股价飙升，直播占陌陌年收入的80%

  • 市场各大平台主播薪资相比15年大概翻了10倍，头部的游戏主播甚至年收入基本过千万

商业化

直播最开始的商业化其实是在国外的色情市场。（作为产品人，要想知道一个产品会不会被市场接受，最直观的一个判断是，黄赌毒会不会用上，如果会，那么一定是可以成的）

以前色情从业人员还得拍片子，需要花费不少成本（人力、物力、时间等），而后很多女性从业者开始做起了直播，通过聊天间与网友互动，网友评论送礼物等，这就开启了直播商业化的大门。很多从业者做得好的，一个月月入2，3万一点压力都没有。

而在中国，色情市场肯定是不能碰的，于是我们看到了直播行业和棋牌类结合、和游戏结合、和演唱会等活动结合，硬生生催熟了国内直播领域市场。

pc端以YY直播、斗鱼直播为代表。移动端则是以映客、花椒为代表。

这些平台也培养了不少具备才气和人气的主播。

前面也说了，这些平台的头部主播的市场价值在16年直播达到顶峰的时候，身价翻了10倍，这是16年统计的游戏领域的主播大致的薪酬：

这些主播的收入来源一般是什么呢？

  • 评论区用户的打赏

  • 主播人气的增加，带来的增值

  • 代言广告

主要是这3大块。

主播跟直播平台签合同，与平台分利润。以映客为例，平台和主播的抽成比例大概为 7:3，但即便是这样，很多主播一个月还是能月入2，3万，可见这个市场是多么的庞大。（像映客一个邮轮礼物大概是1314块，主播能直接分到400块左右）

再一个，你做的时间长了，人气起来了，那么你的个人品牌价值就会变高，比如跳到其他直播平台，薪酬就会翻或者给一些商家进行代言，商家借助你的人气进行促销，给你提成/报酬等。

那么为什么直播的市场盘子这么大呢？它到底满足了用户的什么需求呢？我们这里看一张图。

在这样一个眼球经济时代，随着用户的可支配时间增多，有大量的用户多了很多空闲时间。

那么对于这些人来说，在直播平台上看主播进行个人才艺表演（女主播一般是唱歌，男主播一般是喊麦），打发时间的同时，娱乐自身就是一种很好的方式。

特别是随着“内容付费”时代的到来，人们愿意为有价值的事物进行消费，这在以前的市场下是无法做到的。（比如直播平台女主播，男性用户消费了对方的美【颜值】，要是还能听到自己喜欢的歌，那么在现如今的市场下，他们是愿意为对方进行付费的）

这里主要说说移动端的代表作映客以及陌陌。

映客

映客早起的先发优势占足了，当时获取流量成本非常低。在直播市场未完全规模化的时候，做了很多打擦边球的内容，给平台带来了大量的男性用户和知名度，这也是最初映客崛起的根基。

映客主打国内一二线城市，是纯素人的全名直播。

成立两年不到，就估值过70亿，不得不说是国内互联网市场的一个奇迹。

但蒙眼狂奔的2年，也给映客埋下了一个很大的隐患，就是平台的忠诚度太低，一二线城市的用户并没有在平台上产生粘性，映客为了快速的增长，并没有把重心放在平台的内容生产上，导致平台主播大量流失，用户没有忠诚度，也就随着主播一起去了其他平台。

随着17年的到来，平台的增长速度越来越慢，到最后形成了烧钱都无法获得用户增长的局面。

我们都知道对于一个互联网产品来说，如果无法持续获得增长，也就代表着被市场接受的程度在下降，意味着产品的模式出了问题。

当然，还有平台爆出的机器人事件，弄虚作假，骗流量、骗用户、刷榜被app store下架等等的新闻，这都给映客带来了不小的负面影响。

映客也想转型，但遇到了很大的问题。市场流量不像最初一样好获取，也无法打擦边球了，花椒背后有360、快手背后有腾讯，淘宝直播背后有阿里，但就是映客没有国内市场巨头的投资，导致流量获取越来越难，孤身一人打天下确实争不过巨头。

所以我们看到了映客以60亿的身价卖给了宣亚国际。

同样遇到了增长困难的，还有陌陌。

陌陌

大家对陌陌熟知的最开始，是因为陌陌被称为“约炮软件”，根本在于陌陌的定位是基于地理位置的陌生人社交。

这让陌陌成了除微信外，活得最惬意，最潇洒的一个社交产品。（在国内，有微信这座大山在，做社交基本是没戏，陌陌硬生生的开辟出了一条陌生人社交的路）

同样的，在后期发展的时候，也遇到了天花板问题，用户增长速度放缓。

于是，陌陌尝试了很多不同的场景，企图通过场景的拓宽来加高自己的天花板，比如拼车、支付、红包等，但效果并不显著，直到唐岩在直播热潮到来的时候，果断的切入了直播市场，拓宽了原有的内容生态。

从原有的陌生人社交场景，拓展到了视频内容生态，这就给原先的平台带来了更加多元化的内容产出形式。

陌陌跟很多直播平台不同的是，它的切入很自然，并不是为了直播而做直播，它并不仅仅依赖于头部的主播（比如上面说的映客，就因为主播的大量流走，导致平台用户流失），它的腰部的直播用户量以及营收占到了50%，这说明陌陌所构建的直播内容生态是良性的，更加均衡。

直播的弊端

直播在16年到达顶峰后，在市场流量被消耗殆尽的情况下，也在走向没落，也就有了我们题目所说的“短视频”的崛起。

直播相比短视频的弊端在哪呢？

  • 直播主要表现形式就是主播和粉丝进行互动，这种形式比较单一，难以形成多元化的内容形式，用户很难在平台上形成粘性，进而产生愿意分享的动作，这也导致了它的可传播性不够高，无法产生爆款视频

  • 直播内容创作门槛高，需要主播漏脸，还得长得好看，需要主播能带动气氛，让用户能愿意给你产生互动，但偏偏很多从事主播的人群的学历、段位等都不是很高，可创造的东西极其有限

  • 用户的辨别成本高，它不像我们看15秒的短视频，一下子就能看出来自己是不是喜欢。直播中你需要很长时间才能看出来平台的主播是否是你所喜欢的，愿意花时间/花精力去看的

所以现在的市场已经进入了短视频的赛道，今日头条下的火山小视频、抖音、musically、还有市场上诸如美拍、猫饼、footage等其他更多的产品，下次我们好好聊聊短视频赛道的相关话题。

不会编程，也能爬虫抓取数据(1)

很多人以为只有学会代码，才能爬虫抓数据，这里给大伙讲解一个不用懂编程，也能爬虫抓数据的教程。

本合集课程会由浅入深，分为3篇。

今天第一篇，先从简单的文本、链接、图片、结构体给大家讲解下具体爬虫的过程。

爬虫课程

需要的工具设备就2样，一个chrome浏览器（当然qq浏览器什么的也可以），一个是webscraper插件，去chrome store下载就可以了（如果有无法翻墙下载的，可以评论留言，我给你发）。

这两样具备了，我们就可以直接开始来进行网站的爬虫教学了。

这里再说一下，webscraper插件装好后，用法是在chrome网站页面里“调出开发者工具”

点击“开发者工具”，就会出现这样的页面：

这就是后面我们爬虫需要在这里进行操作的界面了。

爬文本

我们这里以知乎大v产品经理–刘飞老师为例。

知乎链接地址为：https://www.zhihu.com/people/laufei/posts

我们这里就爬刘飞老师发表的所有文章的标题。

首先在这个界面里，打开“开发者工具”，然后点击web scraper选项，再点击Create new sitemap下拉菜单，选中Create sitemap。

进到创建sitemap界面。

填写好sitemap name（随便填，因为我们爬刘飞老师的文章，就以liufei为例），start URL就填上面提供的地址，然后点击Create sitemap。

之后，再点击 Add new selector，添加新的选择器。

接下来添加新的选择器这一步，就是最重要的一点了。

  • id 这里同样随便填，因为我们抓取的是刘飞老师文章的题目，所以命名为liufei-text

  • Type 这里默认选择的就是Text，我们不用更改，因为我们抓的就是纯文本，所以就选Text。（下拉后我们看到，还有诸如图片image、链接url、结构体element等选项，后面抓其他数据的时候会用到）

  • 然后点击select，我们可以看到界面中出现了选择器，我们这里用鼠标从头去选择刘飞老师文章的前2篇文章标题，这样webscraper会帮我们选择好所有同类型的标题（选择2个可以让webscraper明白是什么样同类型的，如果只选择1个，就不能明白）

  • 然后点击选择器最右边的 Done Selecting，之后它会在Select那一行，自动填充好空白的那个白框。

  • 然后我们勾选 Multiple选项，因为我们爬的不止是一个标题，是有很多个。Regex我们不用去管。

  • Delay（ms）这我们填2000，也就是2s，因为考虑到每个人的网速不一样，webscraper在后台抓取数据的时候，会与网速有关，我们这里设置网络2s延迟，你也可以多设置点。

  • 然后选择 Save selector

这一步添加选择器的步骤就结束了。（这也是我们爬数据最重要的一步）

再接下来，我们选择Create new sitemap左边的 Sitemap选项，下拉框中选择 Scrape。

在下面的界面中，我们不用去动 Request interval 和 page load delay中的数据，直接点击start scraping按钮即可。

之后什么操作都不要动，会自动弹出一个新的窗口，webscraper在后台操作爬数据，如果你关闭了弹出来的窗口，就直接停止了爬虫操作。数据爬取完毕后，窗口会自动关闭。

然后你得到了下面的界面，右边红圈中圈出就是我们爬虫爬出来的刘飞老师在该页面下所有的文章标题（其他页码中的标题没有被抓取出来，接下来后面的文章会提到怎么抓全部页码的数据）

我们也可以把这些数据下载下来，生成excel文件。

点击Sitemap下拉选择项中的 Export data as CSV，然后你就可以得到你下载的数据了。

这次爬取文本数据的教程到这就结束了，爬image、url的操作基本是一样的，后面会讲解爬所有页码文章的方法等。

好好住

发表于 2017-12-21 | 分类于 产品分析 |
字数统计: 841 字 | 阅读时长 ≈ 3 分钟

历史背景

好好住这个平台，老板冯肃原来是个做媒体的，因为最初帮自己家人装修才开始关注室内设计这一块。

后来转向做关于居住、装修的自媒体平台、分享平台。

刚开始的图基本都是从HOUZZ、Pinterest上当下来的，所以很多刚开始用好好住平台的用户都感觉，还不如直接上HOUZZ看看来的实在。

这也导致了一个结果就是好好住最初的图片风格都是差不多类似的，完全是根据“冯肃”作为创始人的喜好来的，因此也能理解。

还有好好住的文章，也是从HOUZZ上直接翻译过来的。

现阶段

架不住人家创始人真的是把这个当作一个大事去做，亲自去设计装修过几款房子，大量收集这方面的图片和信息，整理转化成家居信息，传播给有需求的人群。

现在，“冯肃”俨然已经成为了家居界的大IP。

很多人认为好好住平台上的图片“美观”大于“实用”，这也是很多第一次接触这平台的人群对它的第一印象，平台的风格过于统一，没有多样性。

但就我个人认为，能作出好的图片+文字本就已经不容易了，真的做过自媒体的人应该能明白，这种找图片、修图片、写文字、修文稿的过程是多么的艰辛，很多人只简单的看到了一篇800字的文章加上一张图片，就理所应当的认为做这个工作背后不需要花费多少功夫，这就是想当然了。

特别是对于这么一个跨界的人来说，我就更佩服了。

延展

到了17年，好好住的平台已经发展出“话题”、“tag标签”、“专题”等各种功能，在家居这个平台上，我也把玩了其他几款app，“蜗牛”、“装吧”、“土巴兔”、“装修体验馆”、“酷家乐设计师”、“齐家”、“设计家”等。

基本这些“装修类”产品，都是“内容+服务”的模式。在内容上，以图文的形式进行宣传，设计师回复相关问题做服务、平台进行设计视频教学，让用户学习如何设计装修等。

还有个别平台跟“家居店铺”合作，给你提供导流到商铺的链接，从中赚取合作费用等。（也不排除，导流到的店铺就是自己公司开的）

好好住相比于其他app，多了些社区性质，让用户可以沉淀在平台上进行沟通交流，不仅仅是用户之间，还有用户和设计师之间。

其他的平台，很多都是比较粗暴直接。“给你估装修预算”、“预约设计师”、“3D体验”、“2D体验”等，基本上当你注册成了平台用户后，对方有了你的联系方式，就会打电话给你问你是不是有装修计划，blabla。

所以相比之下，相信80后、90后的那批人，会更加欣赏好好住这样比较互联网模式的产品，而不是“土巴兔”、“装吧”，这样相对来说还是比较传统的产品。

产品经理的未来

发表于 2017-12-21 | 分类于 产品经理 |
字数统计: 1,565 字 | 阅读时长 ≈ 5 分钟

先来看看通过招聘网站爬虫（后期也会大概写一些关于爬虫类的教学文章），得到的现阶段市面上互联网各大岗位的热度图：

可以很明显的看到，产品的岗位热度在下降，而运营岗位的热度已经超过了产品。

原因是什么呢？

其实也很简单，就是这几年来产品市场已经越发规范起来，市场上产品同质化现象越来越严重，很多坑、很多规范都由前人走出来了，产品经理依然重要，但纯粹靠产品功能远胜于同类竞品，已经几乎不可能了。

支付宝这个产品做得好吗？淘宝这个产品做的好吗？做的不好，每天吐槽的人多如牛毛。但你用吗？你肯定用，你嘴上骂着马云，手上却一点都没闲着，天天支付宝抢红包，淘宝商品一个接一个。

ofo这个产品做的好吗？也不好，没有GPS，就算找到也大机率是坏的。但你用吗？你照样用，因为你家周边全是这些小黄车，你不用这个你用哪个？随着酷骑、小蓝的相继倒闭，用户钱都提不出来，你不剩下只能用ofo和摩拜吗？

这已经成了市场上的一个常态，真的能靠产品这个层面取胜的例子已经越来越少，市面上真正产品驱动型的公司也越来越少，腾讯、网易、知乎等等，越来越多的公司开始着重市场、着重运营。

因此，你可以看到很多公司里的运营和bd的作用越发的凸显了，特别是那些与线下结合越来越紧密的项目的运营/bd，他们的市场价值也越来越高。

大家都知道产品经理是一个，不用写代码、不用跑业务、不用做设计，从工作内容粗略来看，就是个出原型、对接团队的角色。

市场给你开还算过得去的薪水，是因为你的价值体现在工作的背后，是通过挖掘出领先的产品定义、产品策略、产品架构，打开一个新的市场，或是大幅度超越对手，让自己处于领先的地位，从而占领市场。

从这个角度来说，产品经理拿高薪是成立的。但我们都知道，这一点已经越来越不容易做到。真正能打开市场的即使真的是产品的功劳，那估计也不一定是你的，是你们公司的创始人的，他才是真正意义上的“产品经理”。

所以，你看到了马化腾说自己是腾讯的第一位产品经理，周鸿祎说自己是产品经理。（这也是很多对产品了解的不深刻的人，一看这些大佬标榜自己是产品经理，就想着赶紧也入这个坑，幻想着财富自由、名满天下的一天到来……）

但中国还能再出一个马化腾，再出一个“红衣主教”周鸿祎吗？

难。

那这个时候，我们作为产品经理，该何去何从？

我个人觉得有3点是可以关注的。

产品+运营，二者兼备

自媒体时代到来后，你看到各大领域都产生了很多大v。

咪蒙靠着“毒舌”，红透自媒体圈。助理月薪5万，《喜马拉雅》开课，短短2、3天，卖了300多万。多少人人前骂着咪蒙，人后马不停蹄的付款、还拉着自己的朋友去听？（你看，是不是跟上面说的支付宝/淘宝，是一样的？）

以前，谁知道咪蒙？

所以，你可以看到在这个时代，一个好的高阶运营在市场上是多么吃香。

那么我们作为产品，除了要懂开发的边界、懂android/ios设计规范，如果再懂一些深入到市场上的运营策略，是不是会让自己与他人拉开差距？（当然，运营是个很大的概念，这个可以单独再拎出来写几篇文章了）

深入到某个垂直领域，To B

前几年，微信火起来的时候，市场崇尚“产品驱动”，那时候，有 用户体验、产品思维的产品经理有着极大的市场，所以所有的产品经理都All in到To C端，因为这可以最大化的接触到用户，最大化的与“用户体验”打交道…

现在呢，随着线上的流量几经耗尽，我们看到了越来越多的企业开始着重部署线下的生意。

腾讯投资“永辉超市”，星巴克上海开烘焙工坊…

所以，未来的产品经理，如果可以耕深某一个垂直领域，与线下结合，做To B的生意，一定会比现在单纯做线上的To C产品的天花板要高很多。

人工智能

我从不否认，我们做互联网的就是要尽自己最大的可能靠近风口，这是可以最大化成功的方式。

从Alpha go围棋接连挑落李世石、柯洁后，我们看到了人工智能时代的到来。

前段时间，看到“猿题库”的CEO张小龙说，200万年薪都没留住一个“算法工程师”，想想看吧，你做产品有听过身边产品经理年薪200万的吗？

下一个10年就是人工智能的时代，这是毋庸置疑的，任何还坚守To C的产品经理都会被市场所淘汰。

  • 我们看到了google告诉全世界公司的战略已经转为 ai first。

  • 我们看到了没落的百度，all in ai后，股价大涨，重新被市场接纳。

  • 我们看到了阿里成立“达摩院”…

那么作为产品经理的我们，难道不应该顺势而上，站在潮流的最前沿，迎接下一个10年吗？（人工智能也是一个很大的话题，以后有机会也会写一些文章）
