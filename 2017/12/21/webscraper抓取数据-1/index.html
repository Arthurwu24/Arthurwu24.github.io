不会编程，也能爬虫抓取数据(1)

发表于 2017-12-21 | 分类于 爬虫抓数据 | | 浏览 次
字数统计: 1,082 字 | 阅读时长 ≈ 4 分钟

很多人以为只有学会代码，才能爬虫抓数据，这里给大伙讲解一个不用懂编程，也能爬虫抓数据的教程。

本合集课程会由浅入深，分为3篇。

今天第一篇，先从简单的文本、链接、图片、结构体给大家讲解下具体爬虫的过程。

爬虫课程

需要的工具设备就2样，一个chrome浏览器（当然qq浏览器什么的也可以），一个是webscraper插件，去chrome store下载就可以了（如果有无法翻墙下载的，可以评论留言，我给你发）。

这两样具备了，我们就可以直接开始来进行网站的爬虫教学了。

这里再说一下，webscraper插件装好后，用法是在chrome网站页面里“调出开发者工具”

点击“开发者工具”，就会出现这样的页面：

这就是后面我们爬虫需要在这里进行操作的界面了。

爬文本

我们这里以知乎大v产品经理–刘飞老师为例。

知乎链接地址为：https://www.zhihu.com/people/laufei/posts

我们这里就爬刘飞老师发表的所有文章的标题。

首先在这个界面里，打开“开发者工具”，然后点击web scraper选项，再点击Create new sitemap下拉菜单，选中Create sitemap。

进到创建sitemap界面。

填写好sitemap name（随便填，因为我们爬刘飞老师的文章，就以liufei为例），start URL就填上面提供的地址，然后点击Create sitemap。

之后，再点击 Add new selector，添加新的选择器。

接下来添加新的选择器这一步，就是最重要的一点了。

  • id 这里同样随便填，因为我们抓取的是刘飞老师文章的题目，所以命名为liufei-text

  • Type 这里默认选择的就是Text，我们不用更改，因为我们抓的就是纯文本，所以就选Text。（下拉后我们看到，还有诸如图片image、链接url、结构体element等选项，后面抓其他数据的时候会用到）

  • 然后点击select，我们可以看到界面中出现了选择器，我们这里用鼠标从头去选择刘飞老师文章的前2篇文章标题，这样webscraper会帮我们选择好所有同类型的标题（选择2个可以让webscraper明白是什么样同类型的，如果只选择1个，就不能明白）

  • 然后点击选择器最右边的 Done Selecting，之后它会在Select那一行，自动填充好空白的那个白框。

  • 然后我们勾选 Multiple选项，因为我们爬的不止是一个标题，是有很多个。Regex我们不用去管。

  • Delay（ms）这我们填2000，也就是2s，因为考虑到每个人的网速不一样，webscraper在后台抓取数据的时候，会与网速有关，我们这里设置网络2s延迟，你也可以多设置点。

  • 然后选择 Save selector

这一步添加选择器的步骤就结束了。（这也是我们爬数据最重要的一步）

再接下来，我们选择Create new sitemap左边的 Sitemap选项，下拉框中选择 Scrape。

在下面的界面中，我们不用去动 Request interval 和 page load delay中的数据，直接点击start scraping按钮即可。

之后什么操作都不要动，会自动弹出一个新的窗口，webscraper在后台操作爬数据，如果你关闭了弹出来的窗口，就直接停止了爬虫操作。数据爬取完毕后，窗口会自动关闭。

然后你得到了下面的界面，右边红圈中圈出就是我们爬虫爬出来的刘飞老师在该页面下所有的文章标题（其他页码中的标题没有被抓取出来，接下来后面的文章会提到怎么抓全部页码的数据）

我们也可以把这些数据下载下来，生成excel文件。

点击Sitemap下拉选择项中的 Export data as CSV，然后你就可以得到你下载的数据了。

这次爬取文本数据的教程到这就结束了，爬image、url的操作基本是一样的，后面会讲解爬所有页码文章的方法等。
